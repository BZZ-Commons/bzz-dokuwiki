==== 3. User Story Map ====
Eine User Story Map visualisiert die Aufgaben, die ein Benutzer zur Erreichung seiner Ziele bewältigen muss. Dabei wird ein konkretes Szenario über mehrere Aufgabenschritte abgebildet.\\
Die User Story selber ist eine kurze Beschreibung (ein oder zwei Sätze) einer Tätigkeit. Dabei soll eine atomare Handlung beschrieben werden, d.h. dass die Handlung nicht weiter unterteilt werden kann. Andernfalls würde es sich wieder um eine Aufgabe handeln.
----
//Beispiel 13: aus UX Design//\\
Eine umfassende Aufgabe (Leistung erfassen) wird in ihre Teilschritte (z.B. Leistungserfassung starten) unterteilt. Zu jeder so festgehaltenen Teilaufgabe werden mögliche User Stories ergänzt.\\
