====== LU08d - Security ======

Spring Boot enthält praktischerweise auch einfachere Arten, um mit Passwörtern umzugehen.

==== BCryptPasswordEncoder ====

Beim Aufruf der Methode ''encode'' wird jedes Mal ein neuer zufälliger Salt generiert und verwendet. Dieser Salt wird zusammen mit dem eigentlichen Hash ausgegeben im Format:
$2a$10$[22-Zeichen-Salt][31-Zeichen-Hash]

<code java>
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;

...
BCryptPasswordEncoder encoder = new BCryptPasswordEncoder();
String passwordHash = encoder.encode(password);
</code>

Wird zu einem späteren Zeitpunkt der Hash überprüft, wird entsprechend der korrekte Salt gelesen und verwendet.
<code java>
encoder.matches(plainPassword, passwordHash)
</code>

Dies hat den vorteilhaften Nebeneffekt, das der Salt nicht mehr separat abgespeichert werden muss.

==== Secrets ====
Damit die Credentials für die Datenbank nicht im Repository ersichtlich sind, können diese in der Datei '' application.properties'' unter ''src/main/resources/'' abgelegt werden. Spring Boot erkennt diese Datei und stellt die Werte zur Verfügung.

<code>
server.port=8080
db.username=localuser
db.password=
jwt.secret=
</code>
